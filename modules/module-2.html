✍️ Мистецтво промтів

Як правильно формулювати запити, щоб Cursor робив саме те, що потрібно

⏱ 30 хвилин 📝 5 питань
📋
На нашій кухні це: Промти — це замовлення від клієнта. "Зроби щось смачне" — погане замовлення. "Цезар з куркою, без грінок, соус окремо" — ідеальне. Чим точніше замовлення, тим смачніша страва!

🍽️ Що таке промт?

Уяви, що ти прийшов у ресторан. Ти можеш сказати офіціанту:

«Принесіть мені щось смачне»
«Салат Цезар з куркою, без сухариків, соус окремо»

В першому випадку офіціант принесе щось — може смачне, а може ні. В другому — ти отримаєш саме те, що хотів.

Промт — це твоє замовлення для Cursor AI. Чим точніше ти опишеш, що хочеш — тим краще буде результат. Це найважливіша навичка у роботі з AI.

💡 Промт (prompt) — це просто текстовий запит, який ти пишеш у чат Cursor. Нічого складного — просто правильно сформульоване прохання.

🏗️ Структура хорошого промту

Хороший промт — як добре написане ТЗ. Він має три складові:

1

Контекст — «Над чим я працюю»

Розкажи Cursor, з чим ти маєш справу. Які файли, які дані, який проєкт.

2

Задача — «Що потрібно зробити»

Чітко опиши, який результат ти хочеш отримати. Що створити, змінити, виправити.

3

Формат — «Як це має виглядати»

Опиши бажаний вигляд результату: стиль, кольори, структуру, мову.

Приклад ідеального промту

«В мене є CSV-файл з даними партнера (@partner_data.csv). Створи HTML-звіт з таблицею, яка показує замовлення по тижнях. Використай сучасний дизайн з фірмовими кольорами Bolt (зелений #34D186).»

Бачиш? Тут є все: контекст (CSV-файл з даними), задача (HTML-звіт з таблицею по тижнях), формат (сучасний дизайн, кольори Bolt).

✅ Запам'ятай формулу: Контекст + Задача + Формат = Хороший промт

🎯 Погані vs Хороші промти

Давай порівняємо реальні приклади. Ліворуч — як НЕ треба, праворуч — як ТРЕБА.

Приклад 1: Створення звіту

«зроби звіт»
«Створи HTML-звіт по партнеру Burek за останні 8 тижнів. Дані візьми з @data.csv. Таблиця з колонками: тиждень, замовлення, виручка. Додай графік тренду.»

Приклад 2: Виправлення помилки

«виправ помилку»
«В файлі @report.html графік не показує дані за останній тиждень. Перевір, чи правильно фільтруються дати.»

Приклад 3: Робота з базою даних

«підключи databricks»
«Напиши SQL-запит до Databricks (dbx), який покаже кількість доставлених замовлень по Мальті за останні 7 днів з таблиці fact_order_delivery.»
⚠️ Якщо Cursor зробив не те, що ти хотів — у 90% випадків проблема у промті, а не в AI. Спробуй переформулювати запит точніше.

📎 @-згадки — даємо контекст

В Cursor є суперсила — символ @. Коли ти пишеш промт і додаєш @, ти вказуєш Cursor конкретно, куди дивитись. Це як показати пальцем: «ось цей файл!».

📄

@файл — конкретний файл

Наприклад, @report.html — Cursor відразу «бачить» вміст цього файлу і працює саме з ним.

📁

@папка — ціла папка

Наприклад, @Reports/Burek/ — Cursor переглядає всі файли в папці. Корисно, коли працюєш з проєктом.

🔍

@Codebase — пошук по всьому

Cursor шукає по всіх файлах у твоєму робочому просторі. Як Google, але для твоїх файлів.

🌐

@Web — пошук в інтернеті

Cursor шукає актуальну інформацію в інтернеті. Корисно для свіжих даних чи документації.

📚

@Docs — офіційна документація

Cursor звертається до офіційної документації інструментів. Наприклад, для Databricks чи Python.

Приклад з @-згадкою

«Подивись на @data.csv і створи такий самий звіт, як у @Reports/Burek/Weekly_Report.html, але для партнера PizzaDay.»

💡 Коли ти набираєш @ у чаті, Cursor покаже список доступних файлів і опцій. Просто починай друкувати назву — і обирай зі списку.

📏 Cursor Rules — навчаємо Cursor ТВОЇМ правилам

Уяви, що у тебе є новий асистент. Перший день — ти пояснюєш йому все: «Відповідай українською», «Завжди фільтруй замовлення зі статусом delivered», «Використовуй кольори Bolt». Другий день — ти повторюєш те саме. Третій — знову…

А тепер уяви, що ти можеш записати ці правила один раз — і асистент буде їх дотримуватись завжди. Це і є Cursor Rules.

Що таке Rules?

Rules — це файли з постійними інструкціями для Cursor. Вони зберігаються у папці .cursor/rules/ у твоєму проєкті і автоматично застосовуються при кожному запиті.

Як створити правило?

1

Відкрий налаштування

Натисни Cmd + Shift + P і набери «Cursor Rules».

2

Створи новий файл правил

Або попроси Cursor: «Створи правило в .cursor/rules/ яке каже завжди відповідати українською».

3

Готово!

Cursor тепер буде дотримуватись твого правила при кожному запиті.

Приклади корисних правил

Правило: мова відповіді
Завжди відповідай українською мовою.
Правило: запити до Databricks
При запитах до Databricks завжди фільтруй order_state = 'delivered'.
Завжди додавай LIMIT для великих таблиць.
Використовуй партиційну колонку order_created_date у WHERE.
✅ Rules = постійні інструкції для асистента. Написав один раз — працює завжди. Не потрібно повторювати кожен раз.

🎮 Три режими роботи Cursor

Cursor має три режими роботи — як три передачі в автомобілі. Кожна для свого.

💬 Ask mode — режим запитань

Відкрий чат: Cmd + L

Це як розмова з розумним колегою. Ти питаєш — він відповідає. Cursor тільки читає файли і відповідає на питання, але нічого не змінює. Безпечний режим для початківців.

Коли використовувати Ask

«Поясни, що робить цей код у @report.html?» або «Яка структура таблиці fact_order_delivery?»

🤖 Agent mode — режим агента

Відкрий чат: Cmd + L, потім обери «Agent» у випадаючому меню

Це справжня магія. Agent може створювати файли, редагувати код, запускати команди в терміналі. Він сам вирішує, що потрібно зробити, і робить це.

Коли використовувати Agent

«Створи HTML-звіт з даних @data.csv» — Agent сам прочитає CSV, напише HTML, додасть стилі і графіки.

⚠️ Agent може змінювати файли! Завжди переглядай зміни перед тим, як прийняти їх. Cursor покаже тобі, що саме він хоче змінити.

📋 Plan mode — режим планування

Agent спочатку складає план, показує його тобі, і тільки після твого підтвердження починає виконувати. Ідеально для складних задач.

Коли використовувати Plan

«Переробити всю систему звітів для 5 партнерів» — Plan спочатку покаже крок-за-кроком що буде робити, а потім ти скажеш «го».

💡 Порада для початківців: починай з Ask mode, щоб зрозуміти як працює Cursor. Потім переходь на Agent для реальної роботи. Plan — для масштабних задач.
