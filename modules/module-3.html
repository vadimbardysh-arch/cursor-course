🔌 MCP — підключення зовнішніх сервісів

Як навчити Cursor працювати зі Slack, Jira, Confluence та іншими інструментами

⏱ 20 хвилин 📝 4 питання
🔌
На нашій кухні це: MCP — це кухонне обладнання. Міксер = Slack, духовка = Jira, блендер = Confluence. Підключаєш в розетку — і шеф-повар може ними користуватися!

🔌 Що таке MCP?

Уяви кухню без обладнання — тільки стіл і ніж. Готувати можна, але повільно і обмежено. А тепер додай міксер, духовку, блендер, мікрохвильовку — і ти можеш готувати будь-що!

MCP — це кухонне обладнання для Cursor. Кожне "обладнання" підключає зовнішній сервіс: Slack, Jira, Confluence тощо. Підключив в розетку — і шеф-повар (AI) може ним користуватися.

💡 MCP розшифровується як Model Context Protocol. Але запам'ятовувати не потрібно. Просто думай про MCP як про обладнання, яке розширює можливості кухні.

Без MCP шеф-повар працює тільки з інгредієнтами, що вже на кухні (файли на комп'ютері). З MCP — він може зателефонувати на склад (Slack), перевірити замовлення (Jira), відкрити книгу рецептів (Confluence) та навіть замовити доставку з іншого ресторану (Firecrawl). Різниця величезна!

🚀 Що вміє MCP?

Ось кілька прикладів того, що можна робити з MCP, не виходячи з Cursor:

💬

Slack — повідомлення

Читати та надсилати повідомлення в каналах Slack. Наприклад: «Надішли в #general повідомлення: Звіт за тиждень готовий».

🎫

Jira — тікети

Створювати нові тікети, шукати існуючі, додавати коментарі. «Створи тікет в Jira: Підготувати звіт для партнера X, пріоритет Medium».

📖

Confluence — документація

Шукати по базі знань компанії. «Знайди в Confluence документацію про програму Bolt Plus».

🌐

Firecrawl — веб-сторінки

Читати будь-яку вебсторінку. «Відкрий цей URL і зроби короткий переказ».

🗄️

Бази даних

Пряме підключення до баз даних без додаткових скриптів.

✅ Головна ідея: MCP дозволяє робити все в одному місці — не перемикаючись між десятком вкладок у браузері.

⚙️ Як налаштувати MCP

Є два способи підключити обладнання на кухню. Перший — найпростіший!

Спосіб 1: MCP Маркетплейс (найпростіший!)

Cursor має вбудований маркетплейс — це як магазин кухонного обладнання прямо біля вашого ресторану. Заходиш, вибираєш, підключаєш — все!

1

Відкрий налаштування Cursor

Натисни Cmd + Shift + P → набери «Cursor Settings». Або: Cursor → Settings → Cursor Settings.

2

Перейди в розділ MCP

У лівій панелі натисни «MCP».

3

Натисни «Add MCP Server»

Відкриється список доступних MCP серверів — це і є маркетплейс. Тут можна знайти Slack, Atlassian, Firecrawl та багато інших.

4

Вибери потрібний і натисни Install

Cursor сам все завантажить та налаштує. Тобі залишиться лише авторизуватися (якщо потрібно).

✅ Маркетплейс — це найлегший шлях! Не потрібно нічого копіювати чи вставляти вручну. Просто вибрав і встановив, як додаток на телефон.

Спосіб 2: Ручна конфігурація (для досвідчених)

Якщо потрібного MCP немає в маркетплейсі, або потрібні особливі налаштування — можна додати вручну:

1

В розділі MCP натисни «Add new MCP server»

Обери режим ручного додавання.

2

Вставити конфігурацію

Кожен MCP має свою конфігурацію у форматі JSON (отримаєш від адміна або з документації). Вставляєш — і сервіс підключено!

💡 Порада: Завжди спочатку перевір маркетплейс — там є більшість популярних MCP. Ручне налаштування потрібне рідко.

💬 Налаштування Slack MCP

Slack — один з найпопулярніших MCP. Після підключення ти зможеш читати і писати повідомлення прямо з Cursor.

Крок за кроком:

1

Відкрий MCP налаштування

Cmd + Shift + P → «Cursor Settings» → MCP

2

Натисни «Add new MCP server»

Обери тип «Command» або вставте конфігурацію JSON.

3

Вставити конфігурацію Slack

Конфігурація виглядає приблизно так:

{
  "mcpServers": {
    "slack": {
      "command": "npx",
      "args": ["-y", "@anthropic/slack-mcp"],
      "env": {
        "SLACK_BOT_TOKEN": "xoxb-your-token-here",
        "SLACK_TEAM_ID": "T12345678"
      }
    }
  }
}
⚠️ SLACK_BOT_TOKEN — це секретний ключ. Його видає адміністратор Slack. Ніколи не діліться токенами з іншими людьми і не публікуй їх.

Після налаштування можеш писати:

Приклад промту зі Slack MCP

«Надішли повідомлення в канал #delivery-reports у Slack: Щотижневий звіт по Мальті готовий, дивіться у Google Drive.»

🎫 Налаштування Atlassian (Jira + Confluence)

Atlassian MCP дає доступ одразу до двох інструментів: Jira (тікети та задачі) і Confluence (документація та база знань).

Конфігурація Atlassian MCP:

{
  "mcpServers": {
    "atlassian": {
      "command": "npx",
      "args": ["-y", "@anthropic/atlassian-mcp"],
      "env": {
        "ATLASSIAN_SITE": "your-company.atlassian.net",
        "ATLASSIAN_USER": "your-email@company.com",
        "ATLASSIAN_API_TOKEN": "your-api-token"
      }
    }
  }
}
💡 API токен для Atlassian можна створити тут: id.atlassian.com → Security → API tokens → Create API token. Або попроси адміна допомогти.

Що можна робити після підключення:

Пошук у Confluence

«Знайди в Confluence документацію про процес онбордингу нових партнерів»

Створення тікету в Jira

«Створи тікет в Jira: Підготувати QBR-звіт для партнера Burek. Пріоритет — High. Дедлайн — п'ятниця.»

Пошук тікетів

«Покажи всі мої відкриті тікети в Jira зі статусом In Progress»

🧩 Інші корисні MCP

Екосистема MCP постійно зростає — з'являються нові плагіни. Ось кілька корисних:

🌐 Firecrawl — читання вебсторінок

Cursor сам по собі не може відкрити сайт і прочитати його вміст. Firecrawl MCP дає йому цю суперсилу. Просто скажи:

Приклад з Firecrawl

«Відкрий сторінку https://food.bolt.eu/en-mt/i/burek і зроби короткий переказ, що це за ресторан»

🗄️ Database MCP — підключення до баз даних

Пряме підключення до баз даних (PostgreSQL, MySQL тощо) без написання скриптів. Зручно для швидких запитів.

🔧 І багато іншого…

Нові MCP з'являються регулярно. Головний принцип: якщо ти часто перемикаєшся між Cursor і якимось сервісом — перевір, може для нього вже є MCP.

✅ MCP екосистема росте щодня. Перевіряй нові плагіни на mcp.so — це каталог усіх доступних MCP.

⚠️ Важливі нюанси

🔐 Токени — це секрети. API-токени для Slack, Jira та інших сервісів — це як паролі. Ніколи не надсилай їх у чат, не публікуй у відкритих файлах, не діліся з колегами.
🔄 MCP не працює? Перший крок — перезапусти Cursor (повністю закрий і відкрий). У 80% випадків це вирішує проблему. Якщо ні — перевір, чи правильний токен.
🔌 Можна підключити кілька MCP одночасно. Наприклад, Slack + Jira + Firecrawl. Cursor буде сам розуміти, який MCP використати для твого запиту.

Типові помилки:

Надіслати токен колезі в Slack: «Ось мій SLACK_BOT_TOKEN: xoxb-123...»
Зберігати токени тільки в налаштуваннях Cursor на своєму комп'ютері
Панікувати, якщо MCP перестав працювати
Перезапустити Cursor, перевірити токен, попросити допомогу в адміна
