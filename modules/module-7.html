🧙 Лайфхаки та просунуті техніки

Хоткеї, секрети ефективної роботи та поради, які зроблять тебе Cursor-ніндзею

⏱ 20 хвилин 📝 5 питань 🎯 Просунутий рівень
💡
На нашій кухні це: Лайфхаки — це секрети шеф-повара! Як правильно тримати ніж, які спеції додавати першими, і як врятувати страву, якщо щось пішло не так.

⌨️ Гарячі клавіші (найголовніше!)

Гарячі клавіші — це як виклик таксі одним натисканням замість пішої прогулянки. Спочатку здається, що і так нормально, а потім не уявляєш, як жив без них. Ось найважливіші для macOS:

🌟 Топ-1 — Чат з AI

Cmd + L — відкрити AI-чат. Це найголовніша комбінація у всьому Cursor. Натискаєш — і можеш одразу писати запит до AI.

✏️ Редагування «на місці»

Cmd + I — інлайн-редагування. Виділи фрагмент коду → натисни Cmd + I → опиши, що хочеш змінити. AI змінить прямо там, без відкриття чату.

🔍 Швидкий пошук файлів

Cmd + P — пошук файлу за назвою. Починаєш набирати ім'я файлу — Cursor знаходить його миттєво. Як Spotlight, але для твого проєкту.

🎛️ Палітра команд

Cmd + Shift + P — пошук будь-якої дії. Не пам'ятаєш, де щось знаходиться? Відкрий палітру і набери, що шукаєш.

📂 Бічна панель

Cmd + B — сховати/показати ліву панель з файлами. Корисно, коли хочеш більше місця для роботи.

⬛ Термінал

Cmd + J — сховати/показати термінал. Потрібно подивитись результат команди? Cmd + J і він з'являється знизу.

💬 Коментарі

Cmd + / — закоментувати/розкоментувати рядок коду. Одне натискання — рядок стає коментарем, ще одне — знову код.

⏪ Скасувати дію

Cmd + Z — скасувати останню зміну. Твій найкращий друг, коли щось пішло не так! Натискай стільки разів, скільки потрібно — він відмотає назад крок за кроком.

💡 Порада: Не заучуй усе одразу. Почни з трьох: Cmd + L (чат), Cmd + P (файли), Cmd + Z (скасувати). Решта прийде з практикою.

🎯 Як давати Cursor кращий контекст

Cursor — це дуже розумний помічник, але він не вміє читати думки. Чим більше контексту ти даєш — тим кращий результат отримуєш. Ось перевірені прийоми:

📑 Відкривай потрібні файли перед запитом

Cursor бачить усі відкриті вкладки. Якщо ти хочеш, щоб він врахував якийсь файл — просто відкрий його. Це як покласти потрібні документи на стіл перед розмовою з колегою.

📎 Використовуй @-згадки

Хочеш вказати на конкретний файл? Набери @ і вибери файл зі списку. Cursor точно зрозуміє, про що мова.

Приклад з @-згадкою
Подивись на @report.html і зміни заголовок
на "Тижневий звіт — квітень 2026"

🔄 Перефразовуй, не повторюй

Якщо Cursor не зрозумів — перефразуй запит іншими словами. Повторювати те саме — як кричати голосніше на людину, яка не розуміє мову. Краще скажи інакше.

❌ Просто повторити:

«Зроби графік»
«ЗРОБИ ГРАФІК»
«Я ж сказав — зроби графік!»

✅ Перефразувати:

«Додай стовпчикову діаграму (bar chart) з бібліотекою Chart.js, яка показує кількість замовлень по днях тижня»

📏 Довгі промпти — це добре!

Не бійся писати довгі запити. Cursor обробляє довгі тексти чудово. Чим більше деталей — тим точніший результат. Це як замовлення їжі: «щось смачне» → невідомо що отримаєш; «маргариту без цибулі з додатковим сиром» → саме те, що хотів.

📋 Давай приклади бажаного результату

Якщо можеш — покажи приклад того, що хочеш отримати. Один приклад вартий тисячі слів.

Промпт з прикладом
Створи таблицю з даними. Ось приклад формату:

| Партнер   | Замовлення | Виручка  |
|-----------|-----------|----------|
| Burek     | 1,234     | €15,678  |
| Ваш Лаваш| 987       | €12,345  |

📏 Cursor Rules — правила для команди

Уяви, що ти наймаєш нового асистента. Першого дня ти пояснюєш правила: «Ми завжди пишемо звіти українською», «Ми використовуємо зелений колір Bolt», «Завжди додавай LIMIT до SQL-запитів». Cursor Rules — це саме такі правила, записані у файл.

Як створити:

У папці твого проєкту створи папку .cursor/rules/ і додай файли з розширенням .mdc. Cursor читає ці файли і дотримується правил автоматично.

Приклади корисних правил:

Мова відповідей
Завжди відповідай українською мовою
Правила для Databricks
При запитах до Databricks завжди:
- Фільтруй order_state = 'delivered'
- Додавай партиційну колонку у WHERE
- Додавай LIMIT, якщо я не сказав інакше
Брендинг звітів
При створенні HTML-звітів використовуй
кольори Bolt: primary #34D186, dark #1A1A2E,
background #F8F9FA
Безпека SQL
Завжди додавай LIMIT до SQL-запитів,
якщо я не вказав інакше
✅ Головна фішка: правила застосовуються до ВСІХ, хто відкриває цей workspace. Написав правила один раз — вся команда працює за єдиними стандартами!

🧩 Skills — нові здібності для Cursor

Якщо Cursor Rules — це «правила поведінки», то Skills — це «навички». Це готові інструкції для виконання складних задач.

💡 Аналогія: Rules — це як сказати офіціанту «я вегетаріанець» (загальне правило). А Skill — це як дати йому рецепт конкретної страви, яку ти хочеш (детальна інструкція для конкретної задачі).

Skills живуть у папці .cursor/skills/ і містять детальні інструкції. Наприклад:

Skill для створення партнерських звітів

Файл з інструкціями, який описує: який шаблон використовувати, звідки брати дані, які кольори та шрифти, як називати файли, куди публікувати. Cursor читає цей файл і виконує все за інструкцією.

Завдяки Skills ти можеш стандартизувати складні процеси: один раз описав — і далі Cursor повторює їх ідеально кожного разу.

🚫 Типові помилки та як їх уникнути

Усі ми робимо помилки, особливо на початку. Ось найпоширеніші та способи їх уникнути:

❌ Помилка: Занадто розмитий промпт

«Зроби звіт» — Cursor не знає: який звіт, для кого, які дані, який формат?

✅ Рішення: Будь конкретним

«Створи HTML-звіт для Burek з даними за останній тиждень: замовлення, виручка, середній чек. З графіком по днях.»

❌ Помилка: Не перевіряти результат

Cursor створив файл — і ти одразу відправляєш його партнеру, не глянувши

✅ Рішення: Завжди переглядай

Відкрий файл, подивись на дані, перевір числа. Cursor дуже точний, але подвійна перевірка ніколи не завадить

❌ Помилка: Паніка при помилці

З'явилось червоне повідомлення про помилку — все, кінець світу! 😱

✅ Рішення: Скопіюй помилку і покажи Cursor

Скопіюй текст помилки → встав у чат → «Виправи цю помилку». Cursor сам розбереться!

❌ Помилка: Не використовувати @-згадки

«Зміни файл зі звітом» — який саме файл? У тебе їх може бути десятки

✅ Рішення: Вказуй файли через @

«Зміни заголовок у @Weekly_Report.html» — Cursor точно знає, про що мова

❌ Помилка: Запускати невідомі команди

Cursor пропонує команду в терміналі, ти натискаєш «Run» не читаючи

✅ Рішення: Спитай, що робить команда

«Поясни, що робить ця команда, перш ніж я її запущу» — безпека понад усе!

🛟 Техніки відновлення (коли щось пішло не так)

Не хвилюйся — навіть якщо здається, що «все зламалось», є кілька рятівних кіл:

⏪ Cmd + Z — твій найкращий друг

Скасовує останні зміни крок за кроком. Натискай стільки разів, скільки потрібно. Це працює в будь-якому відкритому файлі.

🔄 Restore у Cursor

Cursor має вбудовану функцію відновлення файлів. Якщо AI змінив файл і ти натиснув «Accept» — можна повернути попередню версію через Cursor.

📜 Git History

Якщо проєкт на GitHub — вся історія змін збережена. Можна подивитися будь-яку попередню версію файлу і повернутися до неї. Попроси Cursor: «Покажи попередню версію цього файлу з git history».

🔌 Перезапуск Cursor

Якщо Cursor «завис» або поводиться дивно — просто закрий і відкрий його заново. Твої файли нікуди не дінуться — вони зберігаються на Google Drive.

⚠️ Золоте правило: Перед великими змінами завжди проси Cursor зробити коміт. Це як зберегти гру перед босом — якщо щось піде не так, завжди є точка повернення.

✅ Чеклист найкращих практик

✅ Щоразу, коли працюєш з Cursor:

☐ Починай з чіткої цілі — сформулюй, що саме хочеш отримати, перш ніж писати промпт

☐ Давай контекст через @-згадки — вказуй конкретні файли, щоб Cursor точно знав, про що мова

☐ Переглядай план перед виконанням — якщо Cursor показує, що збирається зробити — прочитай і підтверди

☐ Перевіряй результати перед поширенням — відкрий звіт, глянь на дані, переконайся, що все правильно

☐ Зберігай робочі промпти — якщо промпт дав чудовий результат, збережи його у файл для повторного використання

☐ Налаштуй Cursor Rules — для інструкцій, які повторюються з кожним запитом

☐ Тримай workspace організованим — чітка структура папок = Cursor краще розуміє проєкт

🏋️

Практикуйся щодня

Спробуй виконувати хоча б одну робочу задачу на день через Cursor. Спочатку буде незвично, через тиждень — не уявлятимеш, як працював без нього

📚

Збирай бібліотеку промптів

Створи файл «prompt-library.md» і записуй туди промпти, які дали гарний результат. Це твоя особиста колекція рецептів

🤝

Ділись знаннями з командою

Покажи колегам, що вмієш. Навчи їх базових речей. Разом ви зможете автоматизувати набагато більше

🔄

Слідкуй за оновленнями

Cursor оновлюється часто — нові функції з'являються регулярно. Кожне оновлення робить його ще потужнішим

✅ Пам'ятай: Cursor — це інструмент, який стає кращим, чим більше ти його використовуєш. Не тому, що він «навчається» (хоча і це теж), а тому, що ТИ навчаєшся працювати з ним ефективніше. Вперед! 💪
