🗄️ Databricks — робота з даними

Підключення до бази даних компанії, SQL-запити через Cursor, основні таблиці

⏱ 25 хвилин 📝 5 питань у квізі
🏭
На нашій кухні це: Databricks — це величезний оптовий склад продуктів. Мільйони позицій: кожне замовлення, кожен кур'єр, кожен партнер. Шеф-повар знає, як знайти потрібні інгредієнти і замовити доставку на кухню.

1. Що таке Databricks (для нас)?

Уяви собі ВЕЛИЧЕЗНИЙ оптовий склад продуктів, де зберігаються ВСІ інгредієнти компанії — кожне замовлення, кожен кур'єр, кожен партнер-ресторан. На відміну від Excel (маленька кухонна полиця), цей склад може вмістити мільйони позицій і працювати блискавично. 🚀

💡 Як це працює на кухні: Ти кажеш шеф-повару: «Мені потрібні свіжі помідори з Мальти за останній тиждень». Шеф-повар (Cursor) йде на склад (Databricks), знаходить потрібний стелаж, бере інгредієнти і приносить на кухню. SQL-запит — це список покупок для складу.

Ми використовуємо Databricks, щоб:

  • 📊 Витягувати дані для звітів
  • 📈 Аналізувати метрики (замовлення, GMV, середні чеки)
  • 🔍 Шукати інсайти по партнерах, містах, кур'єрах
✅ Головне: Ми самі не пишемо код! Ми просимо Cursor написати SQL-запити за нас. Він підключається до Databricks і повертає готові дані.

2. 🔗 Як працює підключення

Щоб Cursor міг «розмовляти» з Databricks, у нас є спеціальне налаштування:

1

Файл dbx.py

Спеціальний Python-скрипт, який вміє підключатися до Databricks. Думай про нього як про «перекладача» між Cursor та базою даних.

2

Файл .env

Файл з паролями та ключами доступу. Як ключ-картка від оптового складу — без неї охоронець не пустить. Цей файл вже налаштований, тобі нічого змінювати не потрібно.

3

Магічне слово dbx

Коли ти пишеш «dbx» у своєму промті, Cursor розуміє, що потрібно звернутися до Databricks. Це як код для замовлення доставки зі складу — без нього шеф-повар навіть не спробує йти на склад!

⚠️ Важливо: Файл .env — це секретний файл з паролями. Ніколи не відправляй його нікому і не копіюй у чат. Cursor знає, де його знайти, і використовує автоматично.

3. 🪄 Як попросити Cursor витягнути дані

Все дуже просто — достатньо написати промт з магічним словом dbx!

💡 Правило: Якщо в твоєму промті є слово dbx — Cursor автоматично звернеться до Databricks. Якщо слова dbx немає — він навіть не спробує.

4. 📋 Основні таблиці, які ми використовуємо

У нашій «бібліотеці» є різні «полиці» (таблиці). Ось головні, які тобі знадобляться:

🏆 Головна таблиця замовлень

fact_order_delivery

Це ГОЛОВНА таблиця з усіма замовленнями. Кожен рядок — одне замовлення. Тут є дати, суми, країни, партнери, статуси. Використовуй її для більшості запитів.

Схема: ng_delivery_spark.fact_order_delivery

💰 Грошові метрики

etl_delivery_order_monetary_metrics

Фінансові показники по кожному замовленню — GMV, комісії, знижки. Якщо потрібні гроші — іди сюди.

Схема: ng_public_spark.etl_delivery_order_monetary_metrics

🍕 Інформація про партнерів

dim_provider_v2

Дані про ресторани/партнерів — назви, адреси, категорії. Потрібно знайти конкретний ресторан? Шукай тут.

Схема: ng_delivery_spark.dim_provider_v2

🏙️ Міста

dim_delivery_city

Інформація про міста — назви, країни, ID. Корисно, коли потрібно зіставити місто з країною.

Схема: ng_delivery_spark.dim_delivery_city

⭐ Підписки Bolt Plus

fact_user_subscriptions

Дані про підписників Bolt Plus — хто підписався, коли, в якій країні. Для аналізу підписок.

Схема: core_models_spark.fact_user_subscriptions

🇲🇹 Для Мальти: код країни — 'mt', ID міст — 324 (Valletta region) та 831 (Gozo region). Але назва колонки з країною різна в різних таблицях! Дивись таблицю нижче.

Як фільтрувати по країні в різних таблицях

Колонки по таблицях
  • fact_order_delivery → city_country_code = 'mt'
  • etl_delivery_order_monetary_metrics → country = 'mt'
  • dim_provider_v2 → country_code = 'mt'
  • etl_eater_cohorts_lcs_weekly_by_city → country_code = 'mt'
  • etl_incentives_provider_targeting_features → provider_country_code = 'mt'
  • Таблиці кур'єрів / кошиків → немає колонки країни, фільтруй по city_id IN (324, 831)

5. ⚠️ Важливі правила (запам'ятай!)

Cursor напише SQL за тебе, але є кілька правил, які варто знати. Якщо щось піде не так — перевір ці пункти:

🚫 ЗАВЖДИ фільтруй order_state = 'delivered'
Без цього фільтра ти порахуєш скасовані та відхилені замовлення разом з доставленими. Уяви, що рахуєш продажі, але включаєш повернення — цифри будуть неправильні!
🚫 ЗАВЖДИ додавай дату в WHERE
Таблиці мають мільйони рядків. Без фільтра по даті запит буде сканувати ВСЕ і працювати дуже довго (або взагалі зависне). Це як шукати книгу в бібліотеці без каталогу — можна блукати вічно.
⚠️ order_created_date в таблиці monetary metrics — це STRING, а не дата!
Замість DATE_SUB використовуй порівняння рядків: order_created_date >= '2026-01-01'. Cursor зазвичай знає про це, але якщо бачиш помилку — перевір цей момент.
⚠️ Додавай LIMIT для великих запитів
Якщо не впевнений, скільки рядків поверне запит, додай LIMIT 100 або LIMIT 1000. Краще спочатку подивитись на маленьку вибірку, ніж чекати 10 хвилин на мільйон рядків.
💡 Підказка: Назва колонки з країною відрізняється в різних таблицях (дивись секцію вище). Якщо отримуєш помилку «column not found» — скоріш за все, проблема саме в цьому.

6. 🎯 Практичний приклад: від промта до результату

Давай пройдемо весь процес від початку до кінця:

1

Пишемо промт

Формулюємо, що хочемо дізнатися, і не забуваємо про dbx.

2

Cursor генерує SQL

Cursor сам обирає потрібну таблицю, пише запит, і виконує його.

3

Отримуємо результат

Дані з'являються прямо в чаті — у вигляді таблиці або тексту.

Наш промт

«dbx — скільки замовлень було доставлено на Мальті за останні 7 днів? Розбий по днях.»

Cursor згенерує і виконає такий SQL-запит:

SELECT order_created_date, COUNT(*) as orders
FROM ng_delivery_spark.fact_order_delivery
WHERE city_country_code = 'mt'
  AND order_state = 'delivered'
  AND order_created_date >= DATE_SUB(CURRENT_DATE(), 7)
GROUP BY order_created_date
ORDER BY order_created_date

Розберемо, що тут написано (простою мовою):

  • SELECT — «покажи мені» дату та кількість замовлень
  • FROM — «з таблиці» замовлень
  • WHERE — «але тільки ті, де»: країна = Мальта, статус = доставлено, за останні 7 днів
  • GROUP BY — «згрупуй по» днях (кожен день — окремий рядок)
  • ORDER BY — «відсортуй» від найстарішого до найновішого
✅ Тобі не потрібно це запам'ятовувати! Cursor напише це сам. Але тепер ти розумієш, що відбувається «під капотом», і можеш перевірити, чи все правильно.
